04 / Webhooks

For server-side reconciliation (e.g., merging shopping carts), configure a webhook to receive merge events when a cookie-cleared session is mapped to a canonical identity.

{
  "event": "identity_merged",
  "canonical_id": "usr_7f3a9b2c",
  "merged_uid": "new-uuid-from-cleared-browser",
  "prior_uid": "old-uuid-from-yesterday",
  "confidence": "medium",
  "timestamp": "2026-06-23T16:00:00Z"
}
Common Use Cases:
  • E-Commerce: Instantly merge returning guest carts when a user returns after clearing cookies.
  • SaaS Growth: Prevent trial registration abuse by identifying duplicate accounts on the same device.
  • Analytics: Stitch anonymous pre-login behavior with authenticated user history.
05 / Architecture
AnonID utilizes a multi-signal hardware hash (Canvas rendering, WebGL floating point precision, OfflineAudioContext, concurrency locks) to bridge identities. We do not store IP addresses.
