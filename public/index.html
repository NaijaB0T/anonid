1. The 5-Line Integration

Drop this script into the <head> of your website. Replace YOUR_API_KEY with the key emailed to you.

<script src="https://api.anonid.dev/anonid.js" data-key="sk_live_YOUR_API_KEY" async></script>
<script>
  window.addEventListener('anonid:ready', (e) => {
    const { resolved_id, is_returning } = e.detail;
    
    // Example: send to your analytics or backend
    console.log("User canonical ID:", resolved_id);
    if (is_returning) {
      console.log("Welcome back!");
      // loadRecommendations(resolved_id);
      // restoreCart(resolved_id);
    }
  });
</script>

2. The API Response Schema

The e.detail payload object exposes the following schema exactly:

{
  "resolved_id": "usr_7f3a9b2c-canonical",
  "uid": "raw-uuid-from-current-cookie",
  "is_new": false,
  "is_returning": true,
  "confidence": "high",       // "high", "medium", "low", or "new"
  "session_count": 7          // Number of times this hardware footprint was seen
}

3. The Webhook Schema (Optional)

If you want to merge server-side records (like shopping carts) when a cookie-cleared user is stitched back to their old identity, contact support to set up a webhook URL.

We will send a POST request with this payload:

{
  "event": "identity_merged",
  "canonical_id": "usr_7f3a9b2c-canonical",
  "merged_uid": "new-uuid-from-cleared-browser",
  "prior_uid": "old-uuid-from-yesterday",
  "confidence": "medium",
  "timestamp": "2026-06-23T16:00:00Z"
}
Privacy Note: AnonID does not store IP addresses, emails, or PII. We use a multi-signal hardware hash (Canvas, WebGL, AudioContext) to deterministically resolve cross-session identity.
